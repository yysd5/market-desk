## ディレクトリ構成

```
market-desk/
├── index.html            # トップページ（ハブ）
├── indicators/
│   └── index.html
├── sessions/
│   └── index.html
├── lot-calculator/
│   ├── index.html
│   └── docs/
│       └── index.html    # 計算ロジック・前提のドキュメント
└── README.md
```

GitHub Pages は `main` ブランチの `/(root)` を公開ソースにしています。
ディレクトリ名がそのままURLになるため、各ページのファイル名は `index.html` に統一しています。

ページ間のリンクは**相対パス**（`./indicators/` など）で書いています。
これによりプロジェクトサイト（`/market-desk/` 配下）とユーザーサイト（root 直下）のどちらに配置しても壊れません。

## 公開設定

Settings → Pages → Build and deployment

- Source: **Deploy from a branch**
- Branch: **main** / **/(root)**

GitHub Free の場合、Pages を使うにはリポジトリが Public である必要があります。

## デザインの共通化

全ページで以下を共有しています。共通CSSファイルは作らず、各ページに直接埋め込む方針です（1ファイル保存でオフラインでも完結させるため）。

- 配色トークン（`--paper` / `--surface` / `--ink` / `--rule` / `--signal` ほか）とライト・ダークの切替
- 書体：見出し Shippori Mincho B1、本文 Zen Kaku Gothic New、数値 IBM Plex Mono
- 画面上部のサイトバー（トップへの戻りリンク＋テーマ切替）
- テーマの選択は `localStorage` のキー `md-theme` に保存し、**サイト内の全ページで共有**されます。未設定なら端末の設定に追従します

トークン定義とサイトバーを変更する場合は、全ページの `<style>` 冒頭にある「market-desk 共通トークン」ブロックをまとめて差し替えてください。

## 方針

- 依存パッケージを増やさない。単一HTMLで完結させる
- 数値・仕様には一次ソース（統計局・中央銀行・各証券会社の公式ページ）へのリンクを付ける
- 出典を示せない推定値は載せない
